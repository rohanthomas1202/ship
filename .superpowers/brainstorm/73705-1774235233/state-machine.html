Weekly Commit Lifecycle

State machine governing each week's commit cycle

State Machine Flow

Transition Rules

Transition Trigger What Happens
DRAFT → LOCKED Scheduled job: Monday 8:00 AM Snapshot commits. No adds/edits/deletes allowed. Week begins.
LOCKED → RECONCILING Scheduled job: Friday 5:00 PM Unlocks completion fields. User can mark items done/partial/not done + add notes.
RECONCILING → RECONCILED User clicks "Submit Reconciliation" Locks reconciliation. Triggers carry-forward of incomplete items.
RECONCILED → next DRAFT Automatic on reconciliation submit Items with PARTIAL or NOT_DONE status are cloned into next week's DRAFT with carried_from_id set.

Edge Cases

  • Empty DRAFT at lock time — Week still locks (user just has no commits). Manager dashboard flags this.
  • User doesn't reconcile by Sunday — Stays in RECONCILING. Next week's DRAFT is created anyway (without carry-forwards). User can reconcile late.
  • Carried items — Appear in new DRAFT pre-linked to their original outcome. User can edit, re-prioritize, or delete them.
