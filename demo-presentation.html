Root Cause

Test files used as any for every mock, hiding type mismatches between test mocks and production code.

The Fix: Typed Mock Factories

// BEFORE: Type system completely bypassed const mockQuery = vi.mocked(pool.query); // picks void overload! mockQuery.mockResolvedValueOnce({ rows: [{id: '1'}] } as any); // AFTER: Full type safety with proper Mock cast const mockQuery = pool.query as Mock<(...args: unknown[]) => Promise<QueryResult>>; mockQuery.mockResolvedValueOnce(mockQueryResult([{ id: '1', title: 'Test' }])); // Helper that returns properly typed QueryResult<R> function mockQueryResult<R extends QueryResultRow>(rows: R[]): QueryResult<R> { return { rows, rowCount: rows.length, command: 'SELECT', oid: 0, fields: [] }; }

Key Discoveries

1. Yjs CRDT Real-Time Collaboration

api/src/collaboration/index.ts

Server-authoritative model where PostgreSQL is the source of truth. Clients sync via WebSocket, server applies edits to in-memory Y.Doc and broadcasts immediately. Persistence is debounced to every 2 seconds — instant UX, batched writes.

Dual storage: binary Yjs state for fast sync + JSON backup for REST API consumers. If server crashes, last-persisted state wins.

2. Unified Document Model

api/src/db/schema.sql

Single documents table with document_type discriminator and JSONB properties. Issues, projects, sprints, wikis — all same table, same editor, same permission system.

Follows Notion's paradigm: the difference between content types is their properties, not their structure. Enables type conversion without data migration.

3. Expression Indexes vs GIN

api/src/db/migrations/038_jsonb_expression_indexes.sql

Generic GIN index on JSONB indexes all keys — versatile but slow for targeted lookups. B-tree expression indexes on specific paths (with partial filters by document_type) turn sequential scans into index scans.

Critical insight: the index expression must exactly match the query expression, including type casts like ::int.

Speaker Notes

Three things stood out. First, the Yjs CRDT architecture elegantly separates real-time sync from persistence — clients get instant updates while database writes are batched. Second, the Notion-style unified document model keeps everything simple — one table, one editor, one permission system. Third, I learned that generic GIN indexes on JSONB are a performance trap for specific lookups — expression indexes with exact matching are the answer.
